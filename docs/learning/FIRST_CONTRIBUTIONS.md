## Welcome!

Thank you for considering contributing to Refine! This guide will help you make your first contribution.

---

## Good First Issues

Look for issues labeled `good first issue` on GitHub:
https://github.com/refinedev/refine/labels/good%20first%20issue

**Common types:**
- Documentation improvements
- Example updates
- Bug fixes
- Test additions

---

## Quick Start

### 1. Fork & Clone

```bash
# Fork on GitHub, then clone your fork
git clone https://github.com/YOUR_USERNAME/refine.git
cd refine
```

### 2. Install Dependencies

```bash
pnpm install
```

### 3. Create Branch

```bash
git checkout -b docs/improve-readme
```

### 4. Make Changes

Edit files, add features, fix bugs.

### 5. Test

```bash
pnpm test
pnpm lint
```

### 6. Commit

```bash
git add .
git commit -m "docs: improve README clarity"
```

### 7. Push

```bash
git push origin docs/improve-readme
```

### 8. Create PR

Open pull request on GitHub!

## Types of Contributions

### Documentation

**Easy & valuable!**
- Fix typos
- Improve clarity
- Add examples
- Update outdated info

**Files:** `documentation/docs/**/*.md`

---

### Examples

**Great for learning!**
- Add new example
- Improve existing example
- Fix bugs in examples

**Directory:** `examples/`

---

### Tests

**High impact!**
- Add missing tests
- Improve test coverage
- Fix flaky tests

**Files:** `**/*.spec.ts`, `**/*.test.tsx`

---

### Code

**More involved:**
- Fix bugs
- Add features
- Improve performance

**Coordinate in issue first!**

---

## PR Guidelines

**Good PR:**
- Clear title: `feat: add feature` or `fix: fix bug`
- Description explains why
- Tests included
- Linting passes
- One concern per PR

**Bad PR:**
- Vague title: "Updates"
- No description
- Breaks tests
- Multiple unrelated changes
