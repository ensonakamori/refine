### Core Philosophy

**Headless Architecture**
- ğŸ¨ **UI Freedom:** Works with Ant Design, Material UI, Mantine, Chakra UI, TailwindCSS, or your own components
- ğŸ§­ **Router Freedom:** Integrates with React Router, Next.js, Remix, or custom routers
- ğŸ”Œ **Backend Freedom:** Connects to REST, GraphQL, Supabase, or any data source

**Provider Pattern**
- Clean separation of concerns through providers
- Swap implementations without changing app code
- Easy testing and maintenance

**Hook-Based API**
- 30+ React hooks for all operations
- Composable and reusable
- TypeScript-first with excellent inference

## ğŸ§  Key Concepts Quick Reference

### Headless Architecture

**ğŸ§  Mental Model:** Refine provides the "brain" (logic), you provide the "body" (UI)

```tsx
// The <Refine /> component = Configuration, not UI
<Refine
  dataProvider={dataProvider}    // How to fetch data
  authProvider={authProvider}    // How to authenticate
  routerProvider={routerProvider} // How to route
  resources={[...]}              // What to manage
>
  {/* YOUR UI goes here - any framework! */}
  <YourApp />
</Refine>
```

### Provider Pattern

**ğŸ§  Mental Model:** Providers are "adapters" between Refine and your infrastructure

- **dataProvider** - Talks to your API
- **authProvider** - Handles authentication
- **routerProvider** - Manages navigation
- **i18nProvider** - Translates text
- **notificationProvider** - Shows messages
- **accessControlProvider** - Enforces permissions
- **auditLogProvider** - Logs activities
- **liveProvider** - Streams real-time updates

### Resources

**ğŸ§  Mental Model:** Resources are "models" or "entities" in your app

```tsx
resources={[
  {
    name: "products",        // Resource name (matches API endpoint)
    list: "/products",       // List view route
    create: "/products/new", // Create view route
    edit: "/products/:id",   // Edit view route
    show: "/products/:id",   // Show view route
    meta: { label: "Products", icon: <Icon /> }
  }
]}
```

### Hooks

**ğŸ§  Mental Model:** Hooks are "smart helpers" that handle complexity for you

```tsx
// Data fetching
const { data, isLoading } = useList({ resource: "products" });

// Forms
const { formProps, saveButtonProps } = useForm({ resource: "products" });

// Tables
const { tableProps, filters, sorters } = useTable({ resource: "products" });

// Auth
const { mutate: login } = useLogin();

// Navigation
const go = useGo();
```
